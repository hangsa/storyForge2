# StoryForge 1.0 — 产品设计文档

## 核心理念

StoryForge 是一个创意叙事操作系统，帮助作者创作长篇小说（以网文为主要场景）。系统围绕三个核心目标设计：

| 核心目标 | 能力链 | 负责系统 |
|---|---|---|
| 构思足够发散、足够创新 | Creative Chain | CreativeOS + Creative Director |
| 剧情足够反转、足够抓人 | Narrative Chain | StoryOS + ReaderOS + Scene Engine + Style Engine |
| 角色/剧情高度一致 | Consistency Chain | MemoryOS + State Machines |

**设计哲学：** 用规则与数据控制作品的骨架（一致性、节奏、读者体验），用 AI 填充血肉（创意发散、文字写作）。AI 负责创作，系统负责把关。

## 二、CreativeOS — 创意引擎

### 2.1 设计目标

帮助作者从一个模糊的想法出发，系统化地发散出高新颖度、有市场潜力的故事设定。不是替代作者的创意，而是提供结构化的创意工具。

### 2.2 灵感种子库（Idea Pool）

存储和管理所有创意灵感。每个灵感记录其标签、来源、新颖度评分、使用次数。作者可以随时回顾、组合、升级旧灵感。

### 2.3 套路模式库（Trope Pool）

系统化的网文套路知识库。每个套路记录：
- **核心机制：** 这个套路怎么运作（如"系统文"的核心是"主角获得外挂系统辅助成长"）
- **市场饱和度：** 该套路在当前市场的饱和程度（0-1），由人工标注 + 数据爬取每季度更新。低于 0.3 标记为"蓝海"，高于 0.8 标记为"红海"
- **子变体：** 该套路的常见变化形式
- **融合潜力：** 可以和哪些其他套路跨界融合
- **反模式：** 已经烂大街的变体方向，建议避开

### 2.4 套路变异器（Mutation Engine）

给定一个基础套路，自动生成新颖的变体。四种变异操作：

| 操作 | 说明 | 示例 |
|---|---|---|
| **逆转** | 反转核心预设 | 系统文 → "系统其实是骗局，在污染宿主意识" |
| **融合** | 跨体裁嫁接 | 重生 + 刑侦 = "凶手也重生了，且知道主角重生" |
| **升维** | 矛盾升级 | 主角无敌 → "无敌但无法伤害任何人" |
| **打破** | 颠覆读者预期 | 预设的反派 → 转变为守护者 |

每次变异输出：变异后的核心设定、由此产生的核心矛盾、对读者的新奇钩子、设定自洽性检查。

### 2.5 矛盾设定生成器（Contradiction Engine）

强设定的本质是内置矛盾。系统提供五个矛盾模板，帮助作者构建天然具有张力的设定：

| 矛盾模板 | 含义 | 示例 |
|---|---|---|
| **能力 × 限制** | 强大能力有苛刻限制 | 能预知他人死亡，但无法预知自己的 |
| **永恒 × 消逝** | 永恒的事物会消逝 | 不死之身，但记忆每百年清零 |
| **身份 × 秘密** | 双重身份矛盾 | 表面是医生，实为器官贩卖网络的受害者 |
| **目标 × 代价** | 达成目标的代价是目标本身 | 追求力量为了保护，但力量在吞噬人性 |
| **力量即弱点** | 最强的力量恰是最大弱点 | 预知能力越强，越清楚一切不可改变 |

### 2.6 连续发散器（WhatIf Engine）

从一个核心前提出发，递归地进行"如果...会怎样？"的发散思考，生成一棵创意树。默认深度3层、广度4个分支，最多可产生 84 个衍生节点。每个子节点必须是父节点的直接逻辑推论，保证发散的逻辑性而非随意跳跃。

### 2.7 体裁融合器（Genre Fusion Engine）

爆款往往来自两个看似不相关体裁的深度嫁接。体裁融合不是简单的"添加元素"，而是在结构特征层面进行融合——找出两个体裁的矛盾点和共振点。

系统维护每个体裁的结构特征定义：
- **核心机制：** 体裁的叙事驱动力（如重生的"预知+改变命运"）
- **读者承诺：** 该体裁对读者的隐含承诺（如系统文的"稳定升级"、刑侦的"烧脑反转"）
- **兼容体裁：** 自然契合的其他体裁
- **冲突体裁：** 存在天然矛盾的体裁（如系统文×刑侦：系统给答案会破坏悬疑）

当两个体裁存在天然冲突时，系统会设计化解方案。体裁之间的距离越远，融合后的新颖度加成越大。

### 2.8 新颖度评估器（Novelty Evaluator）

从四个维度评估一个创意的市场新颖度（0-100分）：

| 维度 | 权重 | 核心问题 |
|---|---|---|
| **市场饱和度** | 30% | 这个套路在市场上有多泛滥？ |
| **套路相似度** | 25% | 与已知套路有多相似？ |
| **矛盾深度** | 25% | 设定是否包含强内置矛盾？ |
| **讨论潜力** | 20% | 读者会不会为这个设定争论？ |

**综合判定：** ≥80 高新颖度，可推进 | ≥60 中等，建议强化矛盾 | ≥40 偏低，需显著变异 | <40 建议更换核心创意

爆款潜力综合评估：新颖度(40%) + 反转密度(30%) + 讨论潜力(30%)

### 2.9 产出物：故事基因（Story DNA）

CreativeOS 的最终产出是一份结构化的"故事基因"文档，包含：
- **核心矛盾：** 故事的根本张力
- **体裁融合方案：** 主体裁 + 融合体裁 + 嫁接点
- **选定的变异套路：** 从套路库中选取并变异后的套路组合
- **关键创意节点：** WhatIf 发散的最优质分支
- **读者钩子设计：** 主谜团、主承诺、情感核心
- **创意约束条件：** 为确保创意自洽而设定的规则边界

## 三、StoryOS — 叙事资产管理

### 3.1 设计目标

长篇小说最大的挑战不是写不出来，而是写到后期忘了前期埋了什么。StoryOS 将叙事中的"无形资产"——冲突、承诺、谜团、反转、揭晓、目标、读者期待——显式登记、追踪、管理，确保每一个伏笔都被回收，每一条剧情线都有始有终。

### 3.2 七类叙事资产

| 资产类型 | 核心问题 | 关键属性 |
|---|---|---|
| **冲突（Conflict）** | 谁和谁在斗？斗到什么程度了？ | 类型(复仇/爱情/权力/生存/意识形态)、强度、预期解决章节 |
| **承诺（Promise）** | 角色对谁许下了什么？什么时候要兑现？ | 说话人、听众、截止章节、完成还是打破 |
| **谜团（Mystery）** | 什么悬念正在吊着读者？线索够了吗？ | 问题、已给出线索清单、计划揭示章节、揭示后的影响级别 |
| **反转（Twist）** | 什么颠覆正在铺垫？读者现在的预期是什么？ | 类型(身份/背叛/世界真相/能力来源/关系)、铺垫方式、铺垫章节列表 |
| **揭晓（Reveal）** | 什么秘密要揭示？谁知道？读者知道吗？ | 秘密内容、谁知道、读者是否知道、揭示方式 |
| **目标（Goal）** | 角色要达成什么？进展如何？ | 类型(短期/弧线/系列)、进度、受什么阻碍/推动 |
| **读者期待（Expectation）** | 读者现在在期待什么发生？ | 期待内容、是满足还是颠覆、是否逾期 |

### 3.3 叙事资产之间的关联网络

这些资产不是孤立的，它们之间存在天然的因果关系：

```
谜团被揭示 → 反转被触发 → 读者期待被满足
冲突升级   → 目标被阻碍
承诺兑现   → 期待被满足
揭示真相   → 冲突进一步升级
```

系统自动追踪这些关联，当一处发生变化时，自动推导并提醒相关的连锁反应。

### 3.4 全书张力曲线

系统实时计算整本书的张力值（0-100），基于当前活跃叙事资产的综合状态：

- 活跃冲突越多、越激烈 → 张力越高
- 即将到来的反转、揭秘越多 → 张力越高
- 开放谜团越多 → 张力越高
- 积压未兑现的读者期待越多 → 需要处理

**智能预警：**
- 连续 3 章张力低于 30 → "节奏过于平缓，需要引入新冲突或推进悬念"
- 承诺即将超期（还剩 5 章以内）→ "该兑现了，或者给出延期理由"
- 读者期待逾期未兑现 → "读者等太久了，有弃书风险"
- 活跃冲突不足 3 个 → "戏剧性不够，建议从现有谜团中衍生新冲突"
- 谜团超过 10 章未推进 → "读者可能已经忘了这个悬念"

## 四、Scene Engine — 写作执行系统

### 4.1 设计目标

将叙事资产的规划落实到每一幕（Scene）的写作中。每个 Scene 不是随意写的，而是带着明确的叙事任务：要揭示哪条线索？要升级哪个冲突？要给读者什么体验？

### 4.2 Scene 的叙事规格

每个 Scene 在写作前需要明确：

**叙事层：**
- 本幕目标（获取线索？推进关系？揭示信息？）
- 本幕冲突（矛盾的即时表现）
- 情感弧线（从什么情绪 → 经历什么 → 到达什么情绪）
- 叙事角色（小爽点 / 大爽点 / 铺垫 / 过渡 / 悬念结尾）

**读者体验层：**
- 本幕是否有反转？
- 本幕兑现了什么？
- 本幕结尾的钩子是什么？
- 预期读者反应（震惊/紧张/满足/期待/愤怒）

**叙事资产管理：**
- 本幕会涉及哪些叙事资产的变动？（哪个谜团给线索？哪个冲突升级？）
- 需要打什么标记以便系统追踪？

### 4.3 "变化标记"机制

这是整个系统最核心的设计之一。传统 AI 写作的问题是：AI 在文字中隐含了角色关系变化、知识获取、冲突升级，但这些变化散落在文字中，难以被系统识别和追踪。

StoryForge 的做法：写作时，AI 在叙事文本的适当位置嵌入"变化标记"（对读者完全不可见，采用 Markdown 注释语法）。这些标记声明了该位置发生的叙事变化：

- 角色关系改变（好感到冷淡、信任到裂痕）
- 角色获得关键知识
- 冲突升级
- 谜团线索被揭示
- 反转被触发
- 读者期待被兑现
- 角色目标达成阶段性进展

一章写完后，系统自动扫描这些标记，提取所有变化，更新叙事资产库——不需要 AI 再去"阅读理解"自己写了什么，精确、确定、零遗漏。

## 五、MemoryOS — 记忆系统

### 5.1 设计目标

长篇小说写作中，作者需要记住的信息分为不同的"热度"。有些信息每时每刻都要知道（主角现在在哪、当前情绪如何），有些信息偶尔需要查阅（30章前那个配角说了什么），有些信息需要被提醒（这个伏笔好像还没回收）。

MemoryOS 设计了五层记忆，按需提供信息：

| 层级 | 内容 | 使用场景 |
|---|---|---|
| **L0 运行时** | 当前章的即时状态：主角在哪、当前情绪、活跃冲突、即将到来的反转 | 写作每一幕时都必须知道 |
| **L1 热记忆** | 最近5章的完整内容 + 关键细节清单 | 保证近期剧情的连贯性 |
| **L2 温记忆** | 整本书的章摘要树、角色关系图、时间线 | 需要了解大局时查阅 |
| **L3 冷记忆** | 全书所有内容的语义检索库 | 需要找回几十章前的某个细节时搜索 |
| **L4 叙事记忆** | 与 StoryOS 同步的叙事资产摘要 | 写作时了解当前所有活跃的伏笔、冲突、承诺 |

### 5.2 细节防遗忘机制

长篇写作中有一个经典问题：AI 倾向于"遗忘"前几章的小细节——主角左手受过伤、某个配角说过一句意味深长的话、一把钥匙还没用过。

系统每 5 章自动运行一次"关键细节重提取"：扫描最近 5 章，提取所有"可能在后续被遗忘但重要"的细节——角色身上的装备/物品/伤情、角色间未完成的对话/约定、环境中被提及但未探索的线索——生成一份清单，放在 AI 写作上下文的顶部。

## 六、ReaderOS — 读者体验模型

### 6.1 设计目标

作者写完一章后，需要知道读者的阅读体验会是什么样。是紧张刺激想追更？还是平淡拖沓想弃书？ReaderOS 从七个维度量化每章的读者体验，全部基于可计算的指标，不依赖 AI 的"感受"。

### 6.2 七个读者状态指标

| 指标 | 衡量什么 | 核心逻辑 |
|---|---|---|
| **好奇心** | 读者有多想知道接下来发生什么？ | 当前开放谜团的数量和重要性 |
| **张力** | 读者有多紧张？ | 活跃冲突和即将到来的反转 |
| **满足感** | 读者最近爽了几次？ | 近3章兑现的期待和承诺数量 |
| **挫败感** | 主角是不是太惨了？ | 近5章主角目标被阻断的次数 |
| **疲劳度** | 读者是不是太累了？ | 连续高强度内容的持续时间 |
| **追更欲** | 读者会不会点下一章？ | 综合好奇心+张力+满足感+结尾钩子质量 |
| **讨论潜力** | 读者会不会在评论区吵架？ | 道德模糊度+预期打破+争议性 |

### 6.3 自动预警

系统在上述指标触发阈值时自动发出预警：

- **追更欲 < 40：** 严重——读者可能弃书，需检查结尾钩子和近期爽点密度
- **挫败感 > 70：** 高度——主角太惨，读者可能心疼到弃书，需安排胜利或进展
- **疲劳度 > 60：** 中度——持续高能，读者需要喘息，建议安排轻松场景或情感缓冲
- **好奇心低但追更欲尚可：** 中度——读者在"追习惯"而非"追内容"，需尽快引入新谜团
- **张力高但讨论度低：** 轻度——"爽但无争议"型章节，可考虑增加道德模糊元素

## 七、一致性保障系统

### 7.1 设计目标

长篇 AI 写作的最大技术挑战是一致性：第 50 章的角色行为和价值观是否和第 5 章一致？角色的能力是否在不经意间膨胀了？已经解决的冲突是否又被无意中重新激活了？

StoryForge 采用两层一致性保障。

### 7.2 角色一致性

每个重要角色维护一份完整的人物档案：

**核心人格层（不轻易改变）：**
- 信念：角色坚信什么
- 欲望：短期/中期/深层目标
- 恐惧：核心创伤和害怕什么
- 价值观：行为底线

**当前状态层（随剧情更新）：**
- 位置、身体状态、情绪状态
- 与每个重要角色的关系状态
- 正在追求的目标
- 角色知道什么秘密、还不知道什么秘密

**角色弧线：** 角色当前处于哪个发展阶段（如"从复仇者转向守护者"）

**角色声音签名：**
- 说话风格（克制简洁？豪放直接？）
- 思维模式（先分析再决策？冲动行事？）
- **绝对禁忌：** 这个角色绝不会做的事（如"不会在公开场合哭泣"、"不会无原则地原谅"）

**信念变化的严格门槛：** 角色的核心价值观不会因为一件小事改变。系统要求信念变化必须满足：有足够多的独立叙事事件支撑（至少 2 个），且变化由明确的叙事事件驱动（被重要人物背叛、见证不可逆的失去、道德觉醒等），而非随意漂移。

### 7.3 剧情线一致性

**剧情线状态追踪：**

```
创建 → 激活 → 升级 → 解决
              ↘ 放弃（作者主动决定）
```

**断线检测：** 系统自动检测所有活跃但长期未推进的剧情线。超过 10 章未被提及的剧情线标记为"可能断线"，核心剧情线阈值为 5 章。

### 7.4 写作审稿机制

每章每幕写完后，系统自动进行三层审查：

**第一层·事实核查（硬性阻断）：**
- 时间线：角色不能同时出现在两个地方；同章内位置变化必须有合理解释
- 角色知识：角色不能展示他还不该知道的秘密
- 能力上限：角色的能力不能超过设定的天花板（除非有明确的"代价"或"突破"理由）
- 剧情合规：已经解决的冲突不能被直接重新激活

如果事实核查不通过 → 打回重写（最多 3 次）。3 次仍不通过 → 触发熔断机制：强制通过但标记问题，通知作者介入。

**第二层·叙事建议（软性建议，不阻断）：**
- 节奏是否合适
- 情绪是否连贯
- 角色行为是否有足够动机
- 检测角色行为是否与其当前状态不符但没有合理解释

**第三层·风格标签（仅记录）：**
- 是否符合流派风格要求
- 是否触犯角色或流派的禁忌

## 八、Style Engine — 风格系统

### 8.1 设计目标

AI 写作容易"千篇一律"。Style Engine 让作者能够定义和控制作品的风格——从宏观的流派模板到微观的句式规则。

### 8.2 三层风格控制

**L1 流派模板：**
预定义的流派写作规则。以"爽文"为例：
- 平均句长 15-25 字（短句快节奏）
- 段落长度 1-3 句
- 对白占全文 30%-50%
- 内心独白每幕不超过 200 字
- 每 2-3 幕至少安排 1 个爽点
- 每章结尾必须有钩子
- 禁忌：连续 3 段以上纯描写、主角超过 1 章无积极行动、配角表现得比主角更出彩超过连续 2 个场景

**L2 写作公式：**
句子级别的定量规则——句子长度的均值和方差、对白密度、段落长度分布。系统自动检测写出的文字是否符合这些定量指标。

**L3 禁忌检测：**
角色级别和流派级别的"不能做"规则。例如某个角色被设定为"不会在公开场合哭泣"——系统自动扫描文本，如果发现该角色在公开场合哭泣的行为，标记为违规。

### 8.3 风格提取

作者可以提供自己喜欢的参考文本（已有的章节、喜欢的作家作品），系统自动分析并提取风格特征：句长分布、对白比例、段落节奏、章节结尾模式、对白标签偏好、过渡词习惯等。这些特征可以转化为风格规则，应用到后续的写作中。

## 九、AI Agent 协作体系

StoryForge 不是单一 AI，而是六个各司其职的 AI Agent 协作：

| Agent | 职责 | 什么时候工作 |
|---|---|---|
| **Conductor 主编排** | 统筹整个创作流程，决定什么时候该做什么，处理异常情况 | 全程 |
| **Creative Director 创意总监** | 驱动所有创意工具（变异、矛盾、发散、融合），产出故事基因 | 构思阶段 |
| **Planner 规划师** | 将故事基因展开为可执行的章节蓝图 | 规划阶段 |
| **Writer 写手** | 按幕写作，嵌入变化标记，遵循风格约束 | 写作阶段 |
| **Reviewer 审稿** | 三层审查（事实/叙事/风格），决定通过还是打回 | 每幕写完后 |
| **StoryOS Agent 叙事管家** | 扫描变化标记，更新所有叙事资产，检查张力曲线 | 每章写完后 |

### 人工门控

作者不是旁观者。系统在以下节点设置人工审核门控：
- 故事基因产出后——作者确认创意方向
- 章节蓝图产出后——作者审核章节规划
- 系统熔断时——3 次重写仍不通过，通知作者介入
- 随时——作者可以随时查看和手动修改任何叙事资产

## 十一、产品价值总结

StoryForge 解决的是长篇小说创作的三个根本难题：

1. **创意枯竭：** 不是凭空想象，而是从系统化的套路库、矛盾模板、变异操作中生成有根据的创意，确保每次产出都有市场新颖度评估。

2. **叙事失控：** 长篇写作中埋下的伏笔、承诺、冲突、谜团被显式追踪，每一个都有状态、有关联、有截止时间。断线自动检测，逾期自动提醒。

3. **一致性崩塌：** 角色行为受限于其人格档案和当前状态，信念不会随意漂移，能力不会偷偷膨胀。每一幕写完后自动通过事实核查。
