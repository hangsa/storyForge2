# T3.11-T3.12 Stage4 Frontend Integration — Design Spec

> **For agentic workers:** This is the design spec. Use `superpowers:writing-plans` to produce the implementation plan from this spec.

**Goal:** Surface precheck results, creative exemption approval, narrative impact, and SF log suggestions inside Stage4Page without bloating the existing 3-column layout.

**Architecture:** 4 new feature-scoped hooks + 7 new components + 1 Stage4Page rewrite + 1 small backend GET endpoint. Matches existing codebase pattern (`useConductor`, `useCreativeCanvas`).

**Tech Stack:** React 18 + TypeScript + Vite + Vitest + React Testing Library. Material Symbols Outlined icons. Existing `GlassPanel` shared component.

---

## Background

StoryForge v1.7 backend now exposes four features that previously had no UI:

1. **Semantic precheck** (Plan 2) — `precheck_result` is included in `writeScene` response. Lists missing SF_LOG tags that the Tier 3 LLM thinks should be in the scene.
2. **Creative exemption** (Plan 3) — `ExemptionManager` + 5 HTTP endpoints. Writer submits exemption requests; user approves/rejects.
3. **User edit assist** (Plan 4) — `SFLogSuggestionEngine` + 2 HTTP endpoints. After user edits draft, analyze diff → propose SF_LOG tags.
4. **Narrative impact** (existing since v1.6) — `/conductor/analyze-impact` endpoint, currently only accessible via `/project/:id/impact` route.

Currently Stage4Page has a 3-column layout (left = draft canvas, right = scene nav + ReaderOS + formulas + SF log feed + Fact Guard). Adding 4 more panels inline would make the right column very long.

## Design Decisions (from brainstorming 2026-06-27)

| Decision | Choice | Rationale |
|---|---|---|
| Layout | Bottom drawer with 4 tabs | Keeps canvas tall, dedicated home for review work, doesn't bloat existing panels |
| Drawer visibility | Auto-open on event + badge when collapsed | User notices new content immediately; badges prevent missed reviews |
| Exemption UX | Modal for full context | Approval needs more space than inline; full context (rule, draft excerpt, antipatterns) prevents poor decisions |
| SF log UX | Dual-section card list (Deleted Tags / Suggested Tags) | Clear separation between info-only deletions and actionable suggestions |
| Narrative impact | Embedded compact panel inside drawer | User gets summary without leaving Stage4; can click through for full report |
| Architecture | Feature-scoped hooks (B) | Matches existing `useConductor`/`useCreativeCanvas` pattern; single-responsibility; testable in isolation |

```
├────────────────────────────────────────────────────────────┤
│  Controls bar (unchanged)                                  │
├─────────────────────────────────┬──────────────────────────┤
│                                 │  Scene nav (unchanged)   │
│  Draft canvas (unchanged)       │  ReaderOS (unchanged)    │
│                                 │  Formulas (unchanged)    │
│                                 │  SF Log feed (unchanged) │
│                                 │  Fact Guard (unchanged)  │
├─────────────────────────────────┴──────────────────────────┤
│  Stage4Drawer (NEW — peek: 60px / expanded: 360px)         │
│  ┌──────────┬──────────┬───────────┬───────────────────┐  │
│  │Precheck 2│Impact    │Exempt 1   │SF Suggestions     │  │

│  [active tab content]                                      │
└────────────────────────────────────────────────────────────┘
```

### Drawer behavior

- **Peek state** (default when collapsed): 60px tall strip showing 4 tab labels with colored count badges (red for action-required like exemption-pending, blue for informational like precheck-suggestions).
- **Expanded state**: 360px tall, shows active tab content.
- **Auto-open**: when a tab's data changes from empty to populated (e.g., writeScene completes → precheck tab gets suggestions → drawer expands to Precheck tab).
- **User collapse**: clicking the drawer header (or pressing Escape on focused drawer) returns to peek.
- **Auto-expand on mount**: no. Always starts collapsed.

### Visual language

- Drawer header: `GlassPanel` style with bottom-rounded corners (mirrors the top panels).
- Active tab: bold text + accent color underline.
- Tab badge: small pill with number, colored by urgency (red = needs action, blue = informational, gray = empty).
- Drawer slides up/down with a 200ms CSS transition.

## Hooks

All new hooks live under `frontend/src/hooks/`.

### `useStage4Precheck.ts`

```ts
interface PrecheckResult {
  precheck_passed: boolean;
  suggestions: Array<{
    event_type: string;
    location_hint: string;
    suggested_tag: string;
    reason: string;
  }>;
  tokens_used: number;
  skipped_reason?: string;
}

interface UseStage4PrecheckReturn {
  data: PrecheckResult | null;
  loading: boolean;
  error: string | null;
  setData: (data: PrecheckResult) => void;
  clear: () => void;
}
```

State: `data` (PrecheckResult | null), `loading` (bool), `error` (string | null).
`setData` is called by Stage4Page after `writeScene` resolves with the response's `precheck_result`.
No async work — pure state holder.

### `useStage4Impact.ts`

```ts
interface UseStage4ImpactReturn {
  report: ImpactReport | null;
  loading: boolean;
  error: string | null;
  run: () => Promise<void>;
  clear: () => void;
}
```

`run()` calls `api.analyzeImpact(projectId)`. State updates on success/error.

### `useStage4Exemptions.ts`

```ts
interface UseStage4ExemptionsReturn {
  items: ExemptionRequest[];
  loading: boolean;
  error: string | null;
  approve: (id: string) => Promise<void>;
  reject: (id: string, reason: string) => Promise<void>;
  refresh: () => Promise<void>;
}
```

`refresh()` calls `api.listExemptions(projectId, "pending")` (new API method, see Backend section).
`approve(id)` calls `api.approveExemption(projectId, id, approvedBy)` then `refresh()`. `approvedBy` is a per-app identifier — v1 uses a hardcoded `"user_default"` constant (no auth system yet; UI shows the constant as the approver name). When auth is added later, this becomes the current user's ID.
`reject(id, reason)` calls `api.rejectExemption(projectId, id, reason)` then `refresh()`.

### `useStage4Suggestions.ts`

```ts
interface SFLogSuggestion {
  type: "missing" | "modified";
  severity: "warning" | "suggestion";
  event_type: string;
  suggested_tag: string;
  location_hint: string;
  reason: string;
}

interface SFLogDiffReport {
  original_text: string;
  modified_text: string;
  deleted_logs: Array<{ raw_text: string; type: string; id: string }>;
  suggestions: SFLogSuggestion[];
  tokens_used: number;
}

interface UseStage4SuggestionsReturn {
  report: SFLogDiffReport | null;
  loading: boolean;
  error: string | null;
  analyze: (original: string, modified: string, charNames: string[]) => Promise<void>;
  apply: (suggestions: SFLogSuggestion[], currentText: string) => string;
  clear: () => void;
}
```

`analyze()` calls `api.suggestSFLogChanges(projectId, sceneId, original, modified)`.
`apply()` is sync (uses engine logic locally — pure substring insertion, no API call needed). Returns the new text. Stage4Page then calls `api.updateSceneDraft(...)` with the new text and `clear()`s the report.

## API client additions

In `frontend/src/api/client.ts`, add types and methods:

```ts
// Types
export interface ExemptionRequest {
  id: string;
  scene_id: string;
  rule_to_break: { layer: string; rule_id: string; rule_description: string; constraint_type: string };
  creative_intent: string;
  expected_effect: string;
  status: "pending" | "approved" | "rejected";
  requested_by: string;
  requested_at: string;
  approved_by: string | null;
  rejected_reason: string | null;
  outcome: string | null;
}

export interface ExemptionAntipattern {
  rule_id: string;
  creative_intent_pattern: string;
  count: number;
  representative_case: string;
}

export interface PrecheckResult { /* as above */ }
export interface SFLogSuggestion { /* as above */ }
export interface SFLogDiffReport { /* as above */ }

// Methods
listExemptions: (projectId: string, status?: "pending" | "approved" | "rejected") =>
  request<ExemptionRequest[]>("GET", `/api/v1/projects/${projectId}/exemptions?status=${status || "pending"}`),

approveExemption: (projectId: string, id: string, approvedBy: string = "user_default") =>
  request<{ id: string; status: string }>("PUT", `/api/v1/projects/${projectId}/exemptions/${id}/approve?approved_by=${encodeURIComponent(approvedBy)}`),

rejectExemption: (projectId: string, id: string, reason: string) =>
  request<{ id: string; status: string }>("PUT", `/api/v1/projects/${projectId}/exemptions/${id}/reject?reason=${encodeURIComponent(reason)}`),
  // NOTE: backend endpoints take `approved_by` and `reason` as query params, not JSON body.

suggestSFLogChanges: (projectId: string, sceneId: string, original: string, modified: string) =>
  request<SFLogDiffReport>("POST", `/api/v1/projects/${projectId}/scenes/${sceneId}/sf-log-suggestions`,
    { original_text: original, modified_text: modified }),
```

## Backend addition (small)

Currently no HTTP endpoint lists pending exemptions. Add to `backend/api/stage4_writing.py`:

```python
@exemptions_router.get("")
def list_exemptions(project_id: str, status: str = "pending") -> list[dict]:
    """List exemption requests for a project, filtered by status."""
    proj_dir = settings.projects_dir / project_id
    mgr = ExemptionManager(proj_dir)
    items = mgr.list_all()
    out = [asdict(e) for e in items if e.status == status]
    return out
```

15 lines including import. Add a test in `tests/test_exemption.py` for the new endpoint.

## Stage4Page rewrite

The existing 640-line `Stage4Page.tsx` is rewritten to:

1. Keep all existing imports + state + handlers (writeScene, forcePass, skip, loadDraft, draftEdit, advance, etc.).
2. Add 4 new hooks: `useStage4Precheck`, `useStage4Impact`, `useStage4Exemptions`, `useStage4Suggestions`.
3. Add drawer state: `activeTab` (`"precheck" | "impact" | "exemption" | "sf-suggestions" | null`).
4. After `handleWrite` success: call `precheck.setData(response.precheck_result)`.
5. After `handleDraftEditSave` success: if old ≠ new, call `suggestions.analyze(old, new, characterNames)`.
6. Render `<Stage4Drawer>` after the closing `</div>` of the main grid.
7. Wire `activeTab` auto-open: when any tab's count goes from 0 → >0, set `activeTab` to that tab (priority order: exemption > sf-suggestions > precheck > impact).

### File outline (post-rewrite, ~750 lines)

```tsx
// imports
import { useStage4Precheck } from "../hooks/useStage4Precheck";
import { useStage4Impact } from "../hooks/useStage4Impact";
import { useStage4Exemptions } from "../hooks/useStage4Exemptions";
import { useStage4Suggestions } from "../hooks/useStage4Suggestions";
import Stage4Drawer from "../components/stage4/Stage4Drawer";
import PrecheckTab from "../components/stage4/PrecheckTab";
import ImpactTab from "../components/stage4/ImpactTab";
import ExemptionTab from "../components/stage4/ExemptionTab";
import ExemptionCard from "../components/stage4/ExemptionCard";
import ExemptionApprovalModal from "../components/stage4/ExemptionApprovalModal";
import SFSuggestionsTab from "../components/stage4/SFSuggestionsTab";

export default function Stage4Page() {
  // existing state and handlers (unchanged behavior)

  // new hooks
  const precheck = useStage4Precheck();
  const impact = useStage4Impact();
  const exemptions = useStage4Exemptions();
  const suggestions = useStage4Suggestions();

  // new state
  const [activeTab, setActiveTab] = useState<DrawerTab | null>(null);
  const [approvalItem, setApprovalItem] = useState<ExemptionRequest | null>(null);

  // after writeScene success: precheck.setData(response.precheck_result)
  // after draft edit save: suggestions.analyze(old, new, characterNames)
  // on first mount: exemptions.refresh()

  // counts for drawer badges
  const counts = {
    precheck: precheck.data?.suggestions.length ?? 0,
    impact: (impact.report?.items ?? []).filter(i => i.priority === "P0" || i.priority === "P1").length,
    exemption: exemptions.items.length,
    sfSuggestions: suggestions.report?.suggestions.length ?? 0,
  };

  // existing JSX (header, chapter progress, controls, 3-column, circuit breaker modal)
  // NEW at the end:
  return (
    <>
      {/* ...existing JSX... */}
      <Stage4Drawer
        counts={counts}
        activeTab={activeTab}
        onTabChange={setActiveTab}
        onCollapse={() => setActiveTab(null)}
      >
        {...}
      </Stage4Drawer>

      {approvalItem && (
        <ExemptionApprovalModal
          item={approvalItem}
          antipatterns={[]} // fetched lazily
          onApprove={async () => { await exemptions.approve(approvalItem.id); setApprovalItem(null); }}
          onReject={async (reason) => { await exemptions.reject(approvalItem.id, reason); setApprovalItem(null); }}
          onClose={() => setApprovalItem(null)}
        />
      )}
    </>
  );
}
```

## Error handling

- Every hook returns `{ data/report/items, loading, error }`. No thrown promises across the UI boundary.
- All API calls wrapped in try/catch. Failures surface as `error` string in the hook state, which the tab renders as a small banner ("加载失败：LLM 不可用") rather than crashing the page.
- Precheck `skipped_reason` renders as info note (not error): "预检已跳过：no router".
- Drawer never blocks the page. Tabs that fail to load show their error inline.
- Modal (ExemptionApprovalModal) is dismissible via Escape or X button — never trapped.

## Scope Explicitly Excluded

- Drawer drag-to-resize (fixed 60px peek / 360px expanded).
- Drawer state persistence across page navigation.
- Real-time refresh of suggestions on background events.
- Keyboard shortcuts for tab switching.
- Bulk-approve across multiple exemptions.
- Animation timing customization (200ms slide is the spec).
- Dark/light theme variants (inherit from existing GlassPanel).
- Mobile layout (drawer behaves same on all viewports; tested on desktop only).

## Risk Notes

1. **Drawer auto-open might annoy users mid-edit**: mitigated by Escape + click-to-collapse + collapsing returns to peek (not full hide).
2. **Backend `list_exemptions` is a small new endpoint** (~15 lines) but is on the critical path for the ExemptionTab. If backend changes break it, the tab fails silently (empty list).
3. **`useStage4Suggestions.apply()` runs logic locally**: relies on the same substring-insertion logic as the backend `apply_suggestions` endpoint. If they diverge, frontend-applied text differs from server-applied text. Mitigation: apply is followed by a server save (`updateSceneDraft`) — server is authoritative on next read.
4. **Modal Escape-key handling**: must not conflict with browser-level Escape (e.g., fullscreen exit). Use `keydown` listener scoped to the modal element.
5. **Drawer + existing circuit breaker modal**: both are fixed-position overlays. They must not appear simultaneously. Mitigation: circuit breaker modal renders above drawer (z-index 60 vs drawer's 50).
