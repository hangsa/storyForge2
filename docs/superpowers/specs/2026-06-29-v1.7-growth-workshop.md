2. 新增 `CharacterDesigner` Agent（Tier 1）引导讨论节奏/代价/同步性
3. 新增 `GrowthConsistencyChecker` 模块（零 LLM，确定性规则）

## 4. 数据模型（兼容 v1.6 schema）

```python
# backend/growth_curve/workshop/models.py（新增）
class ConsistencyWarning(BaseModel):
    rule_id: str               # "missing_event" / "low_misaligned" / "tight_spacing" / "out_of_range" / "invalid_event_type"
    severity: Literal["error", "warning"]
    stage_index: int | None    # 哪个阶段（None 表示跨阶段）
    chapter_number: int | None
    message: str               # 中文提示
    suggestion: str | None     # 可选修复建议

class WorkshopCheckResult(BaseModel):
    character_id: str
    warnings: list[ConsistencyWarning]
    checked_at: str            # ISO timestamp

class WorkshopAdjustRequest(BaseModel):
    stages: list[GrowthStage]  # 完整替换该角色的 growth_curve.stages

class WorkshopDiscussRequest(BaseModel):
    question: str               # 用户问的具体维度（如"节奏是否合适？"）

class WorkshopDiscussResponse(BaseModel):
    answer: str                 # Tier 1 LLM 回答
    suggestions: list[str]      # 可操作建议
```

`GrowthStage` 字段**不新增**（保持 v1.6 schema 兼容）。

## 5. 一致性检查规则（零 LLM，确定性）

| rule_id | 规则 | 严重级 | 数据来源 |
|---|---|---|---|
| `out_of_range` | `stages[-1].chapter_number > outline.total_chapters` | 🔴 error | outline.total_chapters |
| `invalid_event_type` | `stage.trigger_event_type ∉ TRIGGER_KEYWORDS` | 🔴 error | TRIGGER_KEYWORDS.keys() |
| `missing_event` | 转折点章节在 outline 中无 `registry_changes` 匹配其 trigger_event_type | ⚠️ warning | binder._match_trigger_in_registry_changes |
| `low_misaligned` | `stage_name == "低谷"` 的章节 outline 无高烈度 conflict (intensity=critical/high) | ⚠️ warning | outline registry_changes |
| `tight_spacing` | 两个连续 `stage.chapter_number` 差值 < 2 | ⚠️ warning | stages[i+1] - stages[i] |

`intensity` 取自 `registry_changes.created[i].intensity` 或 `updated[i].new_intensity`。

## 6. API 设计

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/projects/{id}/characters/{cid}/growth/workshop/check` | 确定性检查 → 返回 warnings |
| POST | `/api/v1/projects/{id}/characters/{cid}/growth/workshop/discuss` | Tier 1 Agent 讨论 |
| PUT | `/api/v1/projects/{id}/characters/{cid}/growth/workshop/adjust` | 用户调整后保存（先 check 再写） |

**响应统一格式：** `{"error": False, "code": "OK", "detail": {...}}`（与项目其他 API 一致）

**错误处理：**
- `out_of_range` / `invalid_event_type` → 422，**禁止保存**
- 其他 warning → 200，允许保存但 `warnings` 在响应里
- Tier 1 不可用 → discuss 返回 `{"skipped_reason": "no router"}`，UI 降级为静态提示

## 7. CharacterDesigner Agent

对齐 `backend/agents/creative_director.py` 模式：
- 单文件 + 1 个主类 `CharacterDesigner`
- 主方法 `async def discuss(character, outline, question: str) -> DiscussResponse`
- **不做** prompt 内联，放到 `backend/prompts/character_designer/growth_discuss.yaml`
- Tier 1 配置（model_tiers.yaml 已注册）

**4 维度固定 prompt 模板：**
1. 节奏：`间隔是否均衡？是否给读者消化空间？`
2. 代价：`到达低谷的代价是否有说服力？`
3. 同步：`低谷是否与剧情高潮错位？`
4. 主题：`成长方向是否与故事主题一致？`

## 10. 待对齐问题

1. **拖拽 vs 滑块：** 哪种交互更符合用户习惯？spec 倾向"整数滑块 + 拖拽"双输入，需确认。
2. **Agent 讨论的写入策略：** discuss 仅返回建议、不直接修改 stages？还是要"应用建议"按钮？倾向仅返回建议（用户决策权）。
3. **是否需要 impact_analyzer 集成：** 低谷章节调整后是否触发"叙事影响分析"（已存在的 Stage4 功能）？倾向**不集成**，避免耦合。
4. **角色级 vs 项目级一致性：** 单角色 check 已明确，但跨角色耦合（如"两个主角的低谷章节冲突"）是否要做？倾向 v1.8。
