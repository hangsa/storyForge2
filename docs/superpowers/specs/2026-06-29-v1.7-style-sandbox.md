## 1. 背景与目标

v1.6 已有 Style Engine（L1 genre template + L2 writing formulas + L3 taboo constraints）和 `StyleSandboxPage.tsx` 占位页面（仅 extractStyle）。风格沙盒完整化 = 在现有 Style Engine 之上加"预览渲染通道"，让作者用 ~500 字测试文本预览不同参数组合的效果。

**目标：**
1. 加预览渲染 API（Tier 3 LLM，按参数改写）
2. 参数调节 UI（5 类滑块/输入）
3. 原文 vs 渲染结果并排展示
4. 保存为 `<project_dir>/styles/*.yaml`，在 Stage4 风格下拉中可选

## 4. 参数 schema（与设计文档一致）

```python
# backend/style_engine/sandbox_renderer.py（新增）
class SandboxParams(BaseModel):
    sentence: SandboxSentenceParams
    dialogue: SandboxDialogueParams
    rhythm: SandboxRhythmParams
    density: SandboxDensityParams
    satisfaction: SandboxSatisfactionParams

class SandboxSentenceParams(BaseModel):
    avg_length_range: tuple[int, int]      # [15, 45]
    short_sentence_ratio: float            # 0.3
    paragraph_length_range: tuple[int, int] # [80, 200]

# dialogue / rhythm / density / satisfaction 类似（见设计文档 § F1.7.5）

class PreviewRequest(BaseModel):
    source_text: str                       # ~500 字
    params: SandboxParams
    genre: str = "cool_novel"              # 用于选择 L1 模板

class PreviewResponse(BaseModel):
    rendered_text: str
    source_avg_length: float               # 原文统计（用于前后对比）
    rendered_avg_length: float             # 渲染后统计
    tokens_used: int

class SaveStyleRequest(BaseModel):
    name: str                              # 用户命名（e.g. "爽文快节奏 v1"）
    params: SandboxParams

class SavedStyleConfig(BaseModel):
    name: str
    path: str                              # <project_dir>/styles/{name}.yaml
    params: SandboxParams
    created_at: str
```

## 5. API 设计

| 方法 | 路径 | 说明 |
|---|---|---|
| POST | `/api/v1/projects/{id}/style/sandbox/preview` | 预览渲染（Tier 3） |
| POST | `/api/v1/projects/{id}/style/sandbox/save` | 保存配置 |
| GET | `/api/v1/projects/{id}/style/sandbox/configs` | 已保存配置列表 |

**响应统一格式：** `{"error": False, "code": "OK", "detail": {...}}`

**存储路径：** `<project_dir>/styles/{name}.yaml`（name sanitized：去除 `/` `\` 等）

**错误处理：**
- `source_text` 长度 < 50 或 > 2000 字符 → 422（避免无意义渲染）
- Tier 3 不可用 → 返回 `{"skipped_reason": "no router"}`，UI 提示
- 同名配置已存在 → 422，要求改名（避免覆盖）

## 6. 渲染逻辑（sandbox_renderer.py）

```
source_text + SandboxParams + genre
        │
        ▼
  构建 prompt（YAML 模板）：
    - 角色：你是网文风格改写助手
    - 输入：原文 + 5 类参数（量化描述）
    - 约束：保留内容含义，只调整句式和节奏
    - 输出：纯改写后文本
        │
        ▼
  Tier 3 (Haiku) 调用
        │
        ▼
  后处理：
    - 计算 rendered_avg_length
    - 防御：如果 LLM 返回空/含 markdown 包装 → fallback 重试一次
```

**Token 估算：** 输入 ~800 + 输出 ~600 = ~1400/次，与设计文档 § 六.1 一致。

## 7. Stage4 集成（追加）

**当前 Stage4 风格下拉：** v1.6 已支持 L1 体裁模板，**未支持** `.style.yaml` 自定义。

**v1.7.5 追加：**
- `useStage4Writing` 启动时 GET `/style/sandbox/configs` → 合并到风格下拉
- 用户选择自定义风格 → 写作 prompt 注入 `custom_style_config: SandboxParams`
- Writer Agent 在 system prompt 中追加 5 类参数描述

**注意：** 自定义风格仅作为"风格参考"（参考的句长/对白比例），不会强制约束 Writer 必须遵守（避免与 Fact Guard 冲突）。

## 10. 待对齐问题

1. **5 类参数的可调粒度：** 设计文档全列，但用户可能用不到"爽点密度"——是否提供"预设模板"（如"爽文快节奏 / 严肃慢热 / 悬疑紧凑"）一键应用？倾向**加预设**降低学习成本。
2. **预览结果的微调：** 渲染后用户能否"再改一句话"？倾向 v1.8 加局部编辑，v1.7 仅整体预览。
3. **跨场景引用：** 用户能否从 Stage4 的某个 Scene 直接"载入此 Scene 文本到沙盒"？倾向**支持**（提升体验，无技术风险）。
4. **保存后是否自动设为默认风格：** 设计文档未指定，倾向**不自动**——用户主动在 Stage4 切换。
